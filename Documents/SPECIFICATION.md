# 白い悪魔スロット 仕様書

## 概要

白い悪魔スロットは、`whitedevil.html` を台画面、`index.html` を転送入口、`admin.html` を管理画面、`server.js` をローカル同期サーバーとして構成するブラウザ向けスロットです。基本UIとビジュアルは白い悪魔デザインを維持し、スロットの抽選ロジックと演出仕様は GitHub 版 `ntriziinfo/whitedevil` の `whitedevil.html` に統一した新仕様を反映しています。

## 実行方法

- スタンドアロン表示: `whitedevil.html`
- ローカルサーバー表示: `node server.js`
- 台画面: `http://localhost:8787/?machine=台1`
- 管理画面: `http://localhost:8787/admin.html`

## スロット仕様

- 1回転ごとに `3pt` を差し引きます。従来のBET即AT開始ではなく、通常・高確を回してBONUS確定を目指します。
- 通常時は `NORMAL_CEILING_GAMES = 300G` の天井を持ちます。
- 高確は `HIGH_MODE_GAMES = 10G` です。スイカ、チェリー、ベル、砂時計などを契機に移行抽選します。
- BONUS確定後は次BETで7揃いを表示し、その後にATへ突入します。
- AT中は1SET `settings.stSpins` Gを消化し、終了後は4Gの継続バトルへ移行します。
- 継続バトル終了時に勝利していれば次SETへ進み、敗北時は「ATバトルが終了しました。」の結果画面を表示します。
- BIGはデビルゾーン5Gへ入ります。
- REGはSETストック+1です。
- BAR3は全設定共通の1/8192プレミアフラグで、SET+5と継続率80%以上への昇格です。
- 固定払い出しは `ROLE_PAYOUTS` で管理します。主な値は、BIG/REG/砂時計/ベル `15pt`、BAR揃い `0pt`、リプレイ/スイカ `3pt`、チェリー系 `5pt` です。
- 差枚履歴は `stats.slumpHistory` に最大800点まで保存します。
- 設定別目標機械割表示は、設定1=80%、設定2=88%、設定3=95%、設定4=101%、設定5=107%、設定6=114%です。

## 通常・高確・BONUS状態

- `normalState` が通常時の状態を保持します。
  - `mode`: `normal` または `high`
  - `highRemain`: 高確残りG
  - `sinceBonus`: BONUS間ゲーム数
  - `bonusPending`: BONUS確定後、7揃い待ちかどうか
  - `bonusSource`: BONUS確定契機
- `drawNormalResult()` は通常・高確中の役抽選を担当します。
- `resolveNormalOutcome()` は天井、直撃、高確移行、BONUS確定、7揃い待ちを解決します。
- `applyNormalResult()` は通常時の払い出し、役カウント、高確残り、BONUS確定表示、AT突入表示を更新します。

## 演出仕様

- BIGプレミア演出は `assets/puremia.mp4` を使用します。
- BAR3成立時のレバーオン演出は `assets/bar3_lever_on.mp4` を使用します。
- BELL成立時はAT/バトル/デビルゾーン中のみ、停止前に `assets/zako1.mp4`、各STOP時に `assets/zako1-2.mp4` をリール内で再生します。通常・高確中はボタン帯に被らないよう動画を出しません。
- pt払い出しが発生した場合は `assets/payout.wav` を再生します。
- 継続バトル中は `assets/battle_bgm.wav` をループ再生します。バトルBGMは通常BGM/BAR専用BGMより優先し、次SET・終了・初期化時に停止します。
- 継続バトル導入は `assets/battle/battle.png`、`assets/battle/round_*.png`、`assets/battle/character_*.png` を使用します。
- 終了結果画面背景は `assets/result/aozora.jpg` を使用します。
- リプレイ図柄は `assets/replay.png` を使用します。
- 音声素材は `assets/voice/*.wav` を使用します。
- BGM/効果音/動画音声は内部スケールを通して出力します。初期値はBGM `0.22`、効果音 `0.32` です。通常BGM・BAR専用BGM・バトルBGMは共通の `MATCHED_BGM_GAIN_DB = -4.5dB` に揃えます。

## 管理画面

- `server.js` のSSE/APIを通じて複数台の状態をリアルタイム確認できます。
- 各台の設定更新、個別リセット、全台リセット、デバッグ用の直撃操作に対応します。
- 台画面は `machine` クエリで台IDを指定します。
- 台画面は起動時に `/api/machines/:machineId/commands` のSSEへ接続し、管理画面から送信された設定更新、リセット、オート操作、デバッグ操作を受け取ります。接続が切れた場合は自動再接続します。
- 管理画面の台カードには通常・高確・BONUS確定状態、BIG/REG/総回転数/砂時計/ベル/リプレイ/スイカ/チェリー/損益を表示します。
- デバッグ出目指定には `REPLAY` を含みます。
- 1回転コストは現在 `3pt` 固定で、管理画面では読み取り専用です。

## 既知の制約

- 入力欄やセレクトにフォーカスがない状態で `Space` または `ArrowUp` を押すと、BETと同等に動作する。
- 入力欄やセレクトにフォーカスがない状態で `ArrowLeft` は左リール停止、`ArrowDown` は中リール停止、`ArrowRight` は右リール停止として動作する。

### 4.3 デバッグメニュー

- 液晶領域を短時間に5回クリックすると、非公開デバッグメニューを開閉する。
- デバッグメニューには基本設定、次回成立役の強制、次回プレミアBIGフラグ、即時結果ボタンを含める。
- デバッグメニューには通常・高確中の確認用として、高確10Gへ即時移行する `即高確10G` ボタンを含める。AT中、回転中、BONUS確定中は使用しない。
- 音量設定はデバッグメニューには隠さず、画面右上に固定表示した♪アイコンから常時アクセスできる。

## 5. ゲーム仕様

### 5.1 基本進行

- 通常・高確中からボーナスを抽選し、BATTLE BONUSへ移行する。100ptの開始利用料は廃止し、全回転で1Gあたり3ptを投入する。通常・高確中はベル8pt、スイカ3pt、チェリー各種3pt、リプレイ3ptを払い出し、2連チェリーは出現率に設定差を持たせる。
- AT中にゲームを回し、成立役に応じて払い出し、SETストック、ATゲーム数、デビルゾーンを処理する。
- AT消化後は4Gの継続バトルへ入り、最終Gのレア小役では復活抽選を行う。
- 継続バトル敗北またはAT終了確定時は「ATバトルが終了しました。」を表示し、青空背景のリザルト画面に継続SET数と獲得ptを表示する。
- 終了後にBETから新しいATを開始する場合は、開始前に確認ダイアログを表示する。
- AT/ボーナス中の成立役の払い出しは固定ptで処理する。ベル15pt、チェリー全種5pt、スイカ3pt、リプレイ3pt、7揃い・REG・砂時計は15pt、BAR揃いは0pt。通常・高確中はベル8pt、スイカ3pt、チェリー各種3pt、リプレイ3ptを払い出す。
- 通常時にボーナス確定した場合は、`bonusPending` 状態に入り、内部的に777待ちまたはREG待ちを保持する。777待ちの場合は次ゲームで7揃い停止後すぐATへ突入し、REG待ちの場合はREG揃いで50ptを払い出して天井を300Gへリセットする。
- 有効ラインは上段・中段・下段の3ライン。
- 継続率はAT開始時に設定別テーブルから抽選する。10%/30%/50%/60%/80%を含むテーブルを採用し、50%/60%/70%/80%のみの高継続テーブルは機械割上振れが大きいため不採用とする。

### 5.2 主要役

| 役 | 内容 |
| --- | --- |
| BIG | AT/ボーナス中は15pt払い出し。デビルゾーン5Gへ移行。通常時の777確定後は次ゲームの7揃い停止後すぐATへ突入 |
| REG | `7・7・BAR` で判定。通常・高確中のREG確定後は50pt払い出し、ATへ入らず天井を300Gへリセット。AT/ボーナス中は15pt払い出し、+1SET固定 |
| BAR揃い | 払い出し0pt。プレミア役。全設定共通 `1/8192`、+5SET、以後の継続率を80%へ昇格 |
| ベル | 通常・高確中は8pt払い出し。AT/ボーナス中は15pt払い出し。デビルゾーン中は追加抽選あり |
| リプレイ | `REPLAY・REPLAY・REPLAY` で判定。通常・高確中、AT/ボーナス中とも+3ptを払い出し、1Gの投入ptを相殺する。図柄画像は `assets/replay.png` |
| スイカ | AT/ボーナス中は3pt払い出し。AT中・デビルゾーン中に追加抽選あり |
| チェリー2連 | 通常AT中5%で+1SET。デビルゾーン中25%で+1SET/AT+3G |
| チェリー3連 | 通常AT中50%で+1SET。デビルゾーン中は確定+1SET/AT+5G |
| チェリー全種 | AT/ボーナス中は5pt払い出し |
| 砂時計 | AT/ボーナス中は15pt払い出し。画面表示は `AT+加算分`（通常は `AT+10G`）、カウント対象 |

### 5.3 デビルゾーン

- BIG成立時に5Gのデビルゾーンへ入る。
- デビルゾーン中はATを減算しない。
- 成立役ごとにSETストック、ATゲーム数、デビルゾーンゲーム数の抽選を行う。
- デビルゾーン中は各リール内に `assets/devil_zone_fire.mp4` をループ再生する。炎動画は図柄の背面寄りに半透明で重ね、リール停止、zako動画、プレミア動画、バトル開始演出の視認性を邪魔しない範囲で見える濃さに調整する。
- 超デビルゾーン中は同じ炎動画に紫系の色変換フィルターを適用し、通常デビルゾーンの赤い炎と見分けられるようにする。超デビルゾーン中のリール枠グローも紫系へ切り替える。
- デビルゾーン中の主な追加抽選:
  - 3連チェリー: 確定+1SET、AT+5G
  - 2連チェリー: 25%で+1SET、AT+3G
  - スイカ: 25%で+1SET、AT+2G
  - ベル: 10%で+1SET、AT+1G

### 5.4 設定別機械割

- 10%/30%を含む継続率振り分け、4G継続バトル、成立役ごとの固定払い出し（通常・高確ベル8pt、AT/ボーナス中ベル15pt、チェリー5pt、スイカ3pt、リプレイ3pt、7揃い・REG・砂時計15pt、BAR揃い0pt）、砂時計AT+10G（画面表示はAT+加算分）、REG+1SET固定の仕様を維持し、設定別の小役合算当選率を調整して下記の理論機械割へ寄せる。
- BAR揃い後の継続率80%昇格はプレミア上振れ要素として扱い、全体では設定により約+0.7〜+1.0%程度の影響を想定する。

## 7. 永続化仕様

- `localStorage` を使用する。
- ストレージキーは `white_devil_gorai_rebuild_v2`。旧 `white_devil_shippu_st_continue_v1` の保存データは、確率表示や進行状態が旧仕様と混ざるため読み込まない。
- 保存対象は主に `settings` と `stats`。
- `stats.slumpHistory` に差pt推移を保存する。記録点は初期値、各ゲーム開始時の3pt投入後、各ゲーム結果反映後で、保存点数は最大20000点に制限する。0G/0ptの基準点は保持し、そこから実履歴に沿って線を接続する。
- ページ離脱時に現在の設定と統計を保存する。
- 旧保存データに旧機種名が残っている場合も、読み込み時に表示名を「白い悪魔」へ正規化する。

## 8. 音・演出仕様

- 通常BGM、リール操作音、上乗せ/REG/AT残りボイス、BAR専用BGM、プレミア/バトル/リール動画演出を保持する。小当たり・BIGボーナスの専用告知音は再生しない。
- 音量操作は画面右上の♪アイコンから開く `#audioPanel` で行い、スクロール中も最前面に固定表示する。BGM URLが未設定の場合は `assets/media/normal_bgm.mp3` を内蔵通常BGMとして使用する。
- 初期音量はBGM `0.22`、効果音 `0.32` とし、実出力時にBGM・BAR専用BGM・効果音・動画音声ごとの内部係数をかけて全体音量を抑える。通常BGM・BAR専用BGM・バトルBGMは `MATCHED_BGM_GAIN_DB = -4.5dB` を `dbToGain()` で変換した共通係数で出力する。
- 旧保存設定から読み込む場合は、初回読み込み時にBGM/効果音スライダー値を新しい初期音量以下へ自動調整する。
- pt払い出しが発生した場合は `assets/payout.wav` を再生する。音量は効果音音量に連動し、内部係数 `0.25` で他の効果音とバランスを取る。
- 継続バトル突入時は `assets/battle_bgm.wav` をループ再生する。バトルBGMはBGM音量に連動し、通常BGM・BAR専用BGMと同じ `-4.5dB` 係数で音量差が出すぎないようにする。次SETへ進んだ場合、バトルBGMを停止して通常BGMへ戻す。バトル敗北・終了・初期化時もバトルBGMを停止する。

## 38. 2026-05-19 スランプグラフ・設定別確率の表示保持

- スランプグラフと設定別確率パネルは、開いた後にBET、リール停止、AUTO、SPEED、SKIP、筐体外クリックなどを行っても自動で閉じない。

## 39. 2026-05-19 リール枠画像ON/OFF差し替え

- リール枠のベースは常に `assets/frames/slot_cabinet_frame_data_windows_off.png` を表示する。

## 40. 2026-05-19 COMPLETE本採用

- +2000pt到達でCOMPLETE打ち止めとし、設定変更または管理画面の朝一リセットまで遊技不可にする。
- -2000pt到達後は救済状態に入り、フリーズ/BAR揃い確率を1/2000へ優遇する。
- 救済状態中のAT初当たりは、50%の確率で継続率を50%以上へ補正する。非当選時は通常の設定別継続率テーブルを使用する。
- 管理画面の朝一リセットは、設定を据え置いたままCOMPLETEを解除し、状態・モード・スランプグラフ・小役カウントをリセットして朝一モードへ移行する。
- 朝一モードの初回BIG/REG天井は200G、初回当たり後は通常天井300Gへ戻す。
- 設定6の目標値は115%近辺とし、`GORAI_HIT_RATE[6] = 0.965` を採用する。
- 管理画面の1万回転試算は、押下時に状態・モード・スランプグラフ・小役カウントを0へ戻し、朝一モードから10000Gを高速消化する。
- デバッグ高速回転は1000G/sとし、内部抽選・投入pt・払い出しpt・通常/高確・AT・継続バトル・天井・COMPLETE判定・救済判定・スランプグラフを高速更新する。
- リセット、朝一リセット、設定変更後のリール表示は中央段BAR揃いにする。ただし表示出目を揃えるだけで、BAR揃い役としての成立、払い出し、フリーズ、SET上乗せ、BGM切替は発生させない。
- 実抽選で成立するBAR揃いは中段ライン成立に固定する。上段・下段のBAR並びはBAR揃い役として扱わない。

## 42. 2026-05-19 3pt斜めベルによる設定判別
- AT中のリプレイ3ptの一部を、払い出し3ptの斜めベル（BELL3）へ置換する。
- 15ptベルは横一直線の停止形、3ptベルは右下がり/右上がりの斜め停止形で表示する。
- 3pt斜めベルはリプレイ相当の判別用役とし、轟雷ゾーン中のSET抽選、ATゲーム数加算、ゾーンゲーム数加算、BELL専用演出の対象外とする。
- BELL3の重みは設定1から順に30/45/60/80/100/130とし、同数をREPLAYから差し引く。総重みと3pt払い出し総量は維持するため、純増と平均SETへの影響を抑える。


## 43. 2026-05-19 共通AT性能・超デビルゾーン採用
- AT中の役抽選を全設定でほぼ共通化し、純増と1SETあたり期待獲得ptを揃える。
- AT中REG（7・7・BAR）はデビルゾーン5G突入役とし、従来の固定+1SETは廃止する。
- AT中赤7（7・7・7）は超デビルゾーン5G突入役とする。
- デビルゾーン中は現行の上乗せ設計を基本とし、弱チェリーは低確率上乗せに抑える。
- 超デビルゾーン中は3連チェリー、2連チェリー、スイカ、15ptベルのSET/G数/ゾーン延長期待度をデビルゾーンより高くする。
- ゾーン突入時は `devil` / `super` の種別をATセッション内で保持し、AT中BIG由来の超デビルゾーンでは次ゲーム以降も超デビルゾーン専用の上乗せ抽選を適用する。
- AT中の `7・7・BAR` は通常REGとは分離し、内部役 `DEVIL_ZONE` として扱う。AT中の `7・7・7` は通常BIGとは分離し、内部役 `SUPER_DEVIL_ZONE` として扱う。どちらも出目と15pt払い出しは維持するが、BIG/REGカウントには加算しない。
- AT本編中の超デビルゾーン入口は約1/2000へ重くする。デビルゾーン/超デビルゾーン中は毎ゲーム3ptを払い出し、内部的に7揃いチャレンジを行う。チャレンジ成功時のみ `7・7・7` を表示し、50%で+1SET、50%でAT+10Gを上乗せする。DZ中の成功率は1/12、超DZ中の成功率は50%とする。チャレンジ失敗時は7揃いを表示しない。
- デビルゾーン/超デビルゾーン中はAT本編の減算を停止する。AT中のリール上部表示は通常カウント（AT/BIG/REG）を隠し、`AT残`、`DZ残`、`超DZ残` を表示する。
- 3pt斜めベルはリプレイ相当の設定判別役として維持し、上乗せ対象外とする。
- 弱チェリー（CHERRY_ANY）は設定1から順に3/4/5/6/7/8の重みにし、同じ3pt系リプレイ枠から差し引いて出現率判別要素にする。上乗せ率は全設定共通の低確率にする。
