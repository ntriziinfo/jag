# 白い悪魔スロット 仕様書

## 概要

白い悪魔スロットは、`index.html` を台画面、`admin.html` を管理画面、`server.js` をローカル同期サーバーとして構成するブラウザ向けスロットです。基本UIとビジュアルは白い悪魔デザインを維持し、スロットの抽選ロジックと演出仕様は GitHub 版 `ntriziinfo/whitedevil` の `gorai.html` 由来の新仕様を反映しています。

## 実行方法

- スタンドアロン表示: `index.html`
- ローカルサーバー表示: `node server.js`
- 台画面: `http://localhost:8787/?machine=台1`
- 管理画面: `http://localhost:8787/admin.html`

## スロット仕様

- 1回転ごとに `3pt` を差し引きます。従来のBET即ST開始ではなく、通常・高確を回してBONUS確定を目指します。
- 通常時は `NORMAL_CEILING_GAMES = 300G` の天井を持ちます。
- 高確は `HIGH_MODE_GAMES = 30G` です。スイカ、チェリー、ベル、砂時計などを契機に移行抽選します。
- BONUS確定後は次BETで7揃いを表示し、その後にSTへ突入します。
- ST中は1SET `settings.stSpins` Gを消化し、終了後は4Gの継続バトルへ移行します。
- 継続バトル終了時に勝利していれば次SETへ進み、敗北時は「STバトルが終了しました。」の結果画面を表示します。
- BIGはデビルゾーン5Gへ入ります。
- REGはSETストック+1です。
- BAR3は全設定共通の1/8192プレミアフラグで、SET+5と継続率80%以上への昇格です。
- 固定払い出しは `ROLE_PAYOUTS` で管理します。主な値は、BIG/REG/BAR3/砂時計/ベル `15pt`、リプレイ/スイカ `3pt`、チェリー系 `5pt` です。
- 差枚履歴は `stats.slumpHistory` に最大800点まで保存します。
- 設定別目標機械割は、設定1=80%、設定2=90%、設定3=97%、設定4=101%、設定5=110%、設定6=120%です。

## 通常・高確・BONUS状態

- `normalState` が通常時の状態を保持します。
  - `mode`: `normal` または `high`
  - `highRemain`: 高確残りG
  - `sinceBonus`: BONUS間ゲーム数
  - `bonusPending`: BONUS確定後、7揃い待ちかどうか
  - `bonusSource`: BONUS確定契機
- `drawNormalResult()` は通常・高確中の役抽選を担当します。
- `resolveNormalOutcome()` は天井、直撃、高確移行、BONUS確定、7揃い待ちを解決します。
- `applyNormalResult()` は通常時の払い出し、役カウント、高確残り、BONUS確定表示、ST突入表示を更新します。

## 演出仕様

- BIGプレミア演出は `assets/puremia.mp4` を使用します。
- BAR3成立時のレバーオン演出は `assets/bar3_lever_on.mp4` を使用します。
- BELL成立時はST/バトル/デビルゾーン中のみ、停止前に `assets/zako1.mp4`、各STOP時に `assets/zako1-2.mp4` をリール内で再生します。通常・高確中はボタン帯に被らないよう動画を出しません。
- pt払い出しが発生した場合は `assets/payout.wav` を再生します。
- 継続バトル中は `assets/battle_bgm.wav` をループ再生します。バトルBGMは通常BGM/BAR専用BGMより優先し、次SET・終了・初期化時に停止します。
- 継続バトル導入は `assets/battle/battle.png`、`assets/battle/round_*.png`、`assets/battle/character_*.png` を使用します。
- 終了結果画面背景は `assets/result/aozora.jpg` を使用します。
- リプレイ図柄は `assets/replay.png` を使用します。
- 音声素材は `assets/voice/*.wav` を使用します。
- BGM/効果音/動画音声は内部スケールを通して出力します。初期値はBGM `0.22`、効果音 `0.32` です。

## 管理画面

- `server.js` のSSE/APIを通じて複数台の状態をリアルタイム確認できます。
- 各台の設定更新、個別リセット、全台リセット、デバッグ用の直撃操作に対応します。
- 台画面は `machine` クエリで台IDを指定します。
- 台画面は起動時に `/api/machines/:machineId/commands` のSSEへ接続し、管理画面から送信された設定更新、リセット、オート操作、デバッグ操作を受け取ります。接続が切れた場合は自動再接続します。
- 管理画面の台カードには通常・高確・BONUS確定状態、BIG/REG/総回転数/砂時計/ベル/リプレイ/スイカ/チェリー/損益を表示します。
- デバッグ出目指定には `REPLAY` を含みます。
- 1回転コストは現在 `3pt` 固定で、管理画面では読み取り専用です。

## 既知の制約

- 入力欄やセレクトにフォーカスがない状態で `Space` または `ArrowUp` を押すと、BETと同等に動作する。
- 入力欄やセレクトにフォーカスがない状態で `ArrowLeft` は左リール停止、`ArrowDown` は中リール停止、`ArrowRight` は右リール停止として動作する。

### 4.3 デバッグメニュー

- 液晶領域を短時間に5回クリックすると、非公開デバッグメニューを開閉する。
- デバッグメニューには基本設定、次回成立役の強制、次回プレミアBIGフラグ、即時結果ボタンを含める。
- デバッグメニューには通常・高確中の確認用として、高確10Gへ即時移行する `即高確10G` ボタンを含める。AT中、回転中、BONUS確定中は使用しない。
- 音量設定はデバッグメニューには隠さず、画面右上に固定表示した♪アイコンから常時アクセスできる。

## 5. ゲーム仕様

### 5.1 基本進行

- 通常・高確中からボーナスを抽選し、BATTLE BONUSへ移行する。100ptの開始利用料は廃止し、全回転で1Gあたり3ptを投入する。通常・高確中はベル8pt、スイカ3pt、チェリー各種3pt、リプレイ3ptを払い出し、2連チェリーは出現率に設定差を持たせる。
- AT中にゲームを回し、成立役に応じて払い出し、SETストック、ATゲーム数、デビルゾーンを処理する。
- AT消化後は4Gの継続バトルへ入り、最終Gのレア小役では復活抽選を行う。
- 継続バトル敗北またはAT終了確定時は「STバトルが終了しました。」を表示し、青空背景のリザルト画面に継続SET数と獲得ptを表示する。
- 終了後にBETから新しいSTを開始する場合は、開始前に確認ダイアログを表示する。
- AT/ボーナス中の成立役の払い出しは固定ptで処理する。ベル15pt、チェリー全種5pt、スイカ3pt、リプレイ3pt、7揃い・REG・BAR揃い・砂時計は15pt。通常・高確中はベル8pt、スイカ3pt、チェリー各種3pt、リプレイ3ptを払い出す。
- 通常時にボーナス確定した場合は、`bonusPending` 状態に入り、内部的に777待ちまたはREG待ちを保持する。777待ちの場合は次ゲームで7揃い停止後すぐATへ突入し、REG待ちの場合はREG揃いで50ptを払い出して天井を300Gへリセットする。
- 有効ラインは上段・中段・下段の3ライン。
- 継続率は開始時に抽選し、振り分けは10%継続=15%、30%継続=20%、50%継続=50%、60%継続=10%、80%継続=5%。平均継続率は42.5%。通常・高確ベル8pt導入ぶんを吸収するため、60%比率を下げて低継続側へ再配分する。

### 5.2 主要役

| 役 | 内容 |
| --- | --- |
| BIG | AT/ボーナス中は15pt払い出し。デビルゾーン5Gへ移行。通常時の777確定後は次ゲームの7揃い停止後すぐATへ突入 |
| REG | `7・7・BAR` で判定。通常・高確中のREG確定後は50pt払い出し、ATへ入らず天井を300Gへリセット。AT/ボーナス中は15pt払い出し、+1SET固定 |
| BAR揃い | AT/ボーナス中は15pt払い出し。プレミア役。全設定共通 `1/8192`、+5SET、以後の継続率を80%へ昇格 |
| ベル | 通常・高確中は8pt払い出し。AT/ボーナス中は15pt払い出し。デビルゾーン中は追加抽選あり |
| リプレイ | `REPLAY・REPLAY・REPLAY` で判定。通常・高確中、AT/ボーナス中とも+3ptを払い出し、1Gの投入ptを相殺する。図柄画像は `assets/replay.png` |
| スイカ | AT/ボーナス中は3pt払い出し。AT中・デビルゾーン中に追加抽選あり |
| チェリー2連 | 通常AT中5%で+1SET。デビルゾーン中25%で+1SET/AT+3G |
| チェリー3連 | 通常AT中50%で+1SET。デビルゾーン中は確定+1SET/AT+5G |
| チェリー全種 | AT/ボーナス中は5pt払い出し |
| 砂時計 | AT/ボーナス中は15pt払い出し。画面表示は `AT+加算分`（通常は `AT+10G`）、カウント対象 |

### 5.3 デビルゾーン

- BIG成立時に5Gのデビルゾーンへ入る。
- デビルゾーン中はATを減算しない。
- 成立役ごとにSETストック、ATゲーム数、デビルゾーンゲーム数の抽選を行う。
- デビルゾーン中は各リール内に `assets/devil_zone_fire.mp4` をループ再生する。炎動画は図柄の背面寄りに半透明で重ね、リール停止、zako動画、プレミア動画、バトル開始演出の視認性を邪魔しない範囲で見える濃さに調整する。
- デビルゾーン中の主な追加抽選:
  - 3連チェリー: 確定+1SET、AT+5G
  - 2連チェリー: 25%で+1SET、AT+3G
  - スイカ: 25%で+1SET、AT+2G
  - ベル: 10%で+1SET、AT+1G

### 5.4 設定別機械割

- 継続率の新振り分けを採用したうえで、成立役ごとの固定払い出し（通常・高確ベル8pt、AT/ボーナス中ベル15pt、チェリー5pt、スイカ3pt、リプレイ3pt、7揃い・REG・BAR揃い・砂時計15pt）、砂時計AT+10G（画面表示はST+加算分）、REG+1SET固定の仕様を維持し、設定別の小役合算当選率を調整して下記の理論機械割へ寄せる。
- BAR揃い後の継続率80%昇格はプレミア上振れ要素として扱い、全体では設定により約+0.7〜+1.0%程度の影響を想定する。

| 設定 | 目標機械割 |
| --- | ---: |
| 1 | 80% |
| 2 | 90% |
| 3 | 95% |
| 4 | 100% |
| 5 | 110% |
| 6 | 120% |

## 7. 永続化仕様

- `localStorage` を使用する。
- ストレージキーは `white_devil_gorai_rebuild_v2`。旧 `white_devil_shippu_st_continue_v1` の保存データは、確率表示や進行状態が旧仕様と混ざるため読み込まない。
- 保存対象は主に `settings` と `stats`。
- `stats.slumpHistory` に差pt推移を保存する。記録点は初期値、各ゲーム開始時の3pt投入後、各ゲーム結果反映後で、保存点数は最大800点に制限する。
- ページ離脱時に現在の設定と統計を保存する。
- 旧保存データに旧機種名が残っている場合も、読み込み時に表示名を「白い悪魔」へ正規化する。

## 8. 音・演出仕様

- 通常BGM、リール操作音、上乗せ/REG/AT残りボイス、BAR専用BGM、プレミア/バトル/リール動画演出を保持する。小当たり・BIGボーナスの専用告知音は再生しない。
- 音量操作は画面右上の♪アイコンから開く `#audioPanel` で行い、スクロール中も最前面に固定表示する。
- 初期音量はBGM `0.22`、効果音 `0.32` とし、実出力時にBGM・BAR専用BGM・効果音・動画音声ごとの内部係数をかけて全体音量を抑える。
- 旧保存設定から読み込む場合は、初回読み込み時にBGM/効果音スライダー値を新しい初期音量以下へ自動調整する。
- pt払い出しが発生した場合は `assets/payout.wav` を再生する。音量は効果音音量に連動し、内部係数 `0.25` で他の効果音とバランスを取る。
- 継続バトル突入時は `assets/battle_bgm.wav` をループ再生する。バトルBGMはBGM音量に連動し、内部係数 `0.54` で通常BGMと音量差が出すぎないようにする。次SETへ進んだ場合、バトルBGMを停止して通常BGMへ戻す。バトル敗北・終了・初期化時もバトルBGMを停止する。
- 追加ボイスは効果音音量に連動する。上乗せが+1以上発生した場合は `assets/voice/add.wav`、2リール停止時点で7・7が揃った場合は `assets/voice/reach.wav`、REG確定時は `assets/voice/reg.wav` を再生する。
- AT残り回数が3、2、1、0になったレバーON時に、それぞれ `assets/voice/at_3.wav`、`assets/voice/at_2.wav`、`assets/voice/at_1.wav`、`assets/voice/last.wav` を再生する。
- BIG成立時の一部で `assets/puremia.mp4` をリール全面へ半透明表示するプレミア演出を行う。「激熱」画像は使用しない。
- BAR揃い成立時はレバーON直後に `assets/bar3_lever_on.mp4` をリール全面へ不透明表示し、動画再生終了後に透過して非表示へ戻す。
- BAR揃い後は専用BGM、リール周辺の七色点灯演出、継続率80%状態を維持する。ただし継続バトル中はバトルBGMを優先し、次SETへ進んだら通常BGMへ戻す。
- 継続バトルに入る直前の第三停止時だけ、リール全面にラウンド開始演出を表示する。演出には `assets/battle/battle.png`、現在SET数に対応する `assets/battle/round_*.png`、ランダム選択した `assets/battle/character_*.png` を重ねる。通常は青背景、ストックまたは継続率抽選により継続確定済みの場合は赤背景にする。表示後は次のBET操作まで保持し、通常の `BATTLE` テキストオーバーレイは表示しない。`#effectSkipBtn` で即時非表示にする。
