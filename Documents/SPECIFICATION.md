# 白い悪魔スロット 仕様書

## 1. プロジェクト概要

本プロジェクトは、ストグラ内のRP・配信・ゲーム内ポイント用途を想定した、ブラウザ単体で動作するスロット演出HTMLです。

対象機種は「白い悪魔」です。現在の `gorai.html` は、動作していた安定版へ戻した状態です。外部ビルド工程なしでブラウザから直接表示できます。

現金・換金・景品交換・実利益化を目的とした利用は想定しません。

## 3. 現在の画面仕様

### 3.1 筐体デザイン

- 現在は、画像筐体を全面背景にした版ではなく、CSSで構成された安定版のスロット画面を使用する。
- リール絵柄、音源、演出用画像は `gorai.html` 内に埋め込む。
- `assets/gorai_cabinet.webp` は今後もう一度画像筐体化する場合の元素材として保持する。
- 画像筐体化を行う場合は、ボタン座標、リール座標、クリック領域、初期化処理を検証してから反映する。

### 3.2 表示レイアウト

- `.wrap` 内にヘッダー、スロット筐体、設定・統計カードを配置する。
- `.machine` がスロット筐体全体を表す。
- `.reelArea` 内に3リールを表示する。
- `.playControls` 内にBET、STOP、AUTO、演出SKIP操作を配置する。
- 画面右上に常設固定の♪アイコン `#audioToggleBtn` を最前面レイヤーで配置し、BGM/効果音の音量バーとBGM再生・停止操作を `#audioPanel` に格納する。スクロールしても音量パネルは画面右上に追従する。
- サイド設定カードとデバッグメニューは既存の安定版UIを維持する。
- 画面右上の固定ドックに `#slumpToggleBtn`、`#oddsToggleBtn`、`#audioToggleBtn` を配置する。スランプグラフ、確率表、音量設定は常時表示せず、各アイコンからパネルを開いて確認・操作する。
- `#slumpGraph` はスランプパネル内で表示し、総損益（獲得pt - 投入pt）の推移をスランプグラフとして確認できる。

## 4. 操作仕様

### 4.1 ボタン

| 画面上のボタン | HTML要素 | 機能 |
| --- | --- | --- |
| BET | `#spinBtn` | AT開始、1ゲーム開始、または回転中の全リール停止 |
| 左STOP | `#stop0` | 左リール停止 |
| 中STOP | `#stop1` | 中リール停止 |
| 右STOP | `#stop2` | 右リール停止 |
| AUTO | `#quickAutoBtn` | AT、デビルゾーン、継続バトルを含む遊技中の自動消化と停止の切り替え |
| SKIP | `#effectSkipBtn` | 現在表示中の演出動画を即時非表示にする |
| ♪ | `#audioToggleBtn` | 音量パネルの開閉 |

### 4.2 キーボード

- 入力欄やセレクトにフォーカスがない状態で `Space` または `ArrowUp` を押すと、BETと同等に動作する。
- 入力欄やセレクトにフォーカスがない状態で `ArrowLeft` は左リール停止、`ArrowDown` は中リール停止、`ArrowRight` は右リール停止として動作する。

### 4.3 デバッグメニュー

- 液晶領域を短時間に5回クリックすると、非公開デバッグメニューを開閉する。
- デバッグメニューには基本設定、次回成立役の強制、次回プレミアBIGフラグ、即時結果ボタンを含める。
- 音量設定はデバッグメニューには隠さず、画面右上に固定表示した♪アイコンから常時アクセスできる。

## 5. ゲーム仕様

### 5.1 基本進行

- 通常・高確中からボーナスを抽選し、BATTLE BONUSへ移行する。100ptの開始利用料は廃止し、全回転で1Gあたり3ptを投入する。
- AT中にゲームを回し、成立役に応じて払い出し、SETストック、ATゲーム数、デビルゾーンを処理する。
- AT消化後は4Gの継続バトルへ入り、最終Gのレア小役では復活抽選を行う。
- 継続バトル敗北またはAT終了確定時は「STバトルが終了しました。」を表示し、青空背景のリザルト画面に継続SET数と獲得ptを表示する。
- 終了後にBETから新しいSTを開始する場合は、開始前に確認ダイアログを表示する。
- 成立役の払い出しは固定ptで処理する。ベル15pt、チェリー全種5pt、スイカ3pt、リプレイ3pt、7揃い・REG・BAR揃い・砂時計は15pt。
- 通常時にボーナス確定した場合は、即ATへ入らず `bonusPending` 状態に入り、次ゲームで7揃いを成立させた停止後にATへ突入する。
- 有効ラインは上段・中段・下段の3ライン。
- 継続率は開始時に抽選し、振り分けは10%継続=10%、30%継続=15%、50%継続=50%、60%継続=20%、80%継続=5%。平均継続率は46.5%。

### 5.2 主要役

| 役 | 内容 |
| --- | --- |
| BIG | 15pt払い出し。デビルゾーン5Gへ移行。通常時のボーナス確定後は次ゲームの7揃いでATへ突入 |
| REG | `7・7・BAR` で判定。15pt払い出し、+1SET固定 |
| BAR揃い | 15pt払い出し。プレミア役。全設定共通 `1/8192`、+5SET、以後の継続率を80%へ昇格 |
| ベル | 15pt払い出し。デビルゾーン中は追加抽選あり |
| リプレイ | `REPLAY・REPLAY・REPLAY` で判定。通常・高確・AT/ボーナス中に+3ptを払い出し、1Gの投入ptを相殺する。図柄画像は `assets/replay.png` |
| スイカ | 3pt払い出し。通常AT中・デビルゾーン中に追加抽選あり |
| チェリー2連 | 通常AT中5%で+1SET。デビルゾーン中25%で+1SET/AT+3G |
| チェリー3連 | 通常AT中50%で+1SET。デビルゾーン中は確定+1SET/AT+5G |
| チェリー全種 | 5pt払い出し |
| 砂時計 | 15pt払い出し。画面表示は `ST+加算分`（通常は `ST+10G`）、カウント対象 |

### 5.3 デビルゾーン

- BIG成立時に5Gのデビルゾーンへ入る。
- デビルゾーン中はATを減算しない。
- 成立役ごとにSETストック、ATゲーム数、デビルゾーンゲーム数の抽選を行う。
- デビルゾーン中の主な追加抽選:
  - 3連チェリー: 確定+1SET、AT+5G
  - 2連チェリー: 25%で+1SET、AT+3G
  - スイカ: 25%で+1SET、AT+2G
  - ベル: 10%で+1SET、AT+1G

### 5.4 設定別機械割

- 継続率の新振り分けを採用したうえで、成立役ごとの固定払い出し（ベル15pt、チェリー5pt、スイカ3pt、リプレイ3pt、7揃い・REG・BAR揃い・砂時計15pt）、砂時計AT+10G（画面表示はST+加算分）、REG+1SET固定の仕様を維持し、設定別の小役合算当選率を調整して下記の理論機械割へ寄せる。
- BAR揃い後の継続率80%昇格はプレミア上振れ要素として扱い、全体では設定により約+0.7〜+1.0%程度の影響を想定する。

| 設定 | 目標機械割 |
| --- | ---: |
| 1 | 80% |
| 2 | 90% |
| 3 | 97% |
| 4 | 101% |
| 5 | 110% |
| 6 | 120% |

## 7. 永続化仕様

- `localStorage` を使用する。
- ストレージキーは `white_devil_gorai_rebuild_v1`。
- 保存対象は主に `settings` と `stats`。
- `stats.slumpHistory` に差pt推移を保存する。記録点は初期値、各ゲーム開始時の3pt投入後、各ゲーム結果反映後で、保存点数は最大800点に制限する。
- ページ離脱時に現在の設定と統計を保存する。
- 旧保存データに旧機種名が残っている場合も、読み込み時に表示名を「白い悪魔」へ正規化する。

## 8. 音・演出仕様

- 通常BGM、リール操作音、上乗せ/REG/AT残りボイス、BAR専用BGM、プレミア/バトル/リール動画演出を保持する。小当たり・BIGボーナスの専用告知音は再生しない。
- 音量操作は画面右上の♪アイコンから開く `#audioPanel` で行い、スクロール中も最前面に固定表示する。
- 初期音量はBGM `0.22`、効果音 `0.32` とし、実出力時にBGM・BAR専用BGM・効果音・動画音声ごとの内部係数をかけて全体音量を抑える。
- 旧保存設定から読み込む場合は、初回読み込み時にBGM/効果音スライダー値を新しい初期音量以下へ自動調整する。
- pt払い出しが発生した場合は `assets/payout.wav` を再生する。音量は効果音音量に連動し、内部係数 `0.25` で他の効果音とバランスを取る。
- 継続バトル突入時は `assets/battle_bgm.wav` をループ再生する。バトルBGMはBGM音量に連動し、内部係数 `0.54` で通常BGMと音量差が出すぎないようにする。次SETへ進んだ場合、バトルBGMを停止して通常BGMへ戻す。バトル敗北・終了・初期化時もバトルBGMを停止する。
- 追加ボイスは効果音音量に連動する。上乗せが+1以上発生した場合は `assets/voice/add.wav`、2リール停止時点で7・7が揃った場合は `assets/voice/reach.wav`、REG確定時は `assets/voice/reg.wav` を再生する。
- AT残り回数が3、2、1、0になったレバーON時に、それぞれ `assets/voice/at_3.wav`、`assets/voice/at_2.wav`、`assets/voice/at_1.wav`、`assets/voice/last.wav` を再生する。
- BIG成立時の一部で `assets/puremia.mp4` をリール全面へ半透明表示するプレミア演出を行う。「激熱」画像は使用しない。
- BAR揃い成立時はレバーON直後に `assets/bar3_lever_on.mp4` をリール全面へ不透明表示し、動画再生終了後に透過して非表示へ戻す。
- BAR揃い後は専用BGM、リール周辺の七色点灯演出、継続率80%状態を維持する。ただし継続バトル中はバトルBGMを優先し、次SETへ進んだら通常BGMへ戻す。
- 継続バトルに入る直前の第三停止時だけ、リール全面にラウンド開始演出を表示する。演出には `assets/battle/battle.png`、現在SET数に対応する `assets/battle/round_*.png`、ランダム選択した `assets/battle/character_*.png` を重ねる。通常は青背景、ストックまたは継続率抽選により継続確定済みの場合は赤背景にする。表示後は次のBET操作まで保持し、通常の `BATTLE` テキストオーバーレイは表示しない。`#effectSkipBtn` で即時非表示にする。

AT/ボーナス中のベル揃いが内部成立した回転では、レバーON時に全リールへ `assets/zako1.mp4` を表示し、各リールのSTOPボタンが押されるまで表示と再生を維持する。各STOPボタン停止時に対応リールで `assets/zako1-2.mp4` を再生する。zako動画は約0.58の透過率でリールへ重ね、`#effectSkipBtn` で現在回転中の演出表示をスキップできる。通常・高確中のベルではzako動画を表示しない。

## 12. 2026-05-17 通常・高確・ボーナス構成

- 100ptの開始利用料は廃止し、通常・高確・AT/ボーナス・継続バトルを含む全回転で1Gあたり3ptを投入ptとして加算する。
- 払い出しは倍率ではなく固定ptで処理する。ベル15pt、チェリー全種5pt、スイカ3pt、リプレイ3pt、7揃い・REG・BAR揃い・砂時計15pt。
- 通常時にボーナス確定したら即ATへ移行せず、次ゲームで7揃いをそろえることでATへ突入する。
- 通常中は天井300G。天井到達時はBATTLE BONUS確定。
- 通常中の3連チェリーは50%、2連チェリーは25%でBATTLE BONUS確定。

- 2026-05-17追記: 画面表記は高確へ統一する。通常・高確中の砂時計は通常抽選テーブルから除外し、代わりにリプレイ比率を上げる。小役成立の払い出し音とベルzako演出はAT/ボーナス中のみ発生させ、通常・高確中は表示しない。BAR揃いは通常・高確・AT/ボーナスを問わず常時1/8192の独立抽選とする。
